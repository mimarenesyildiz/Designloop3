# 📊 PROMPT KALİTESİ DEĞERLENDİRMESİ

## Güçlü Yanlar

| # | Nokta | Açıklama |
|---|-------|----------|
| 1 | **Multi-agent pipeline** | Konsept → DNA → Prompt Composer ayrımı iyi bir mimari karar |
| 2 | **DNA kilitleri** | form/material/color DNA'sı granüler kontrol sağlıyor |
| 3 | **Confidence gating** | Vision yanıtlarında 0-1 güven skoru ile otomatik kapsam dışı bırakma |
| 4 | **Unified prompt merkezi** | Tek bir `buildUnifiedDesignPrompt()` ile tüm downstream'e tutarlılık |
| 5 | **JSON schema örnekleri** | Her prompt'un sonunda beklenen JSON şeması var |

## 🔴 Kritik Sorunlar ve İyileştirme Önerileri

### 1. Hardcoded "cafeteria" ve Jenerik Referanslar

**Sorun**: `buildControlledVisualPrompt()` içinde:
- `"cafeteria composition"` → hardcoded
- `"table layout"` → konferans salonunda masa düzeni yok, koltuk sırası var
- `"planters, TV screens"` → konferansta saksı/TV olmayabilir

**Çözüm**: Mekan tipi bilgisini vision analizinden çıkart ve prompt'ta dinamik kullan:
```js
const spaceType = appState.analysisSummary?.spaceType || "interior space";
// "cafeteria" yerine "${spaceType}" kullan
```

### 2. Mock Verinin Proje-Agnostik Olması

**Sorun**: Mock analiz her zaman aynı 5 metraj kalemini döndürür:
- Konferans salonu için "Resepsiyon ön alanı" anlamsız
- 228 koltuk saymak yerine "Sabit Mobilya: 4 adet" diyor
- `baseArea = 32 + files.length * 4` formülü 310 m²'lik salonu 36 m² yapıyor

**Çözüm**: Mock verisini en azından plan dosya adından (regex ile "konferans", "ofis", "lobi" gibi) bağlam-duyarlı hale getir.

### 3. Critique Skoru ve API Şeması Uyumsuzluğu

**Sorun**: 
- Mock: `planCozumu`, `esikDeneyimi`, `isikKalitesi`, `oranOranti`, `sirkulasyon`, `odakNoktasi`, `ergonomi`, `esneklik`, `butunselEtki`
- API: `mekansalOrganizasyon`, `ergonomiEsneklik`, `isikPerformansi`, `malzemeDoku`, `gorselKompozisyon`, `konseptUyumu`, `uretilebilirlik`, `butceVerimliligi`

Mock ve API tamamen farklı skor setleri kullanıyor! UI render edip "designCritique.skorlar" okurken alan eşleşmezse boş veri gösterir.

**Çözüm**: Mock veriyi API ile aynı şemaya getir veya UI'da her iki şemayı da kontrol et.

### 4. Prompt'larda Dil Tutarsızlığı

**Sorun**: 
- Vision ve alternative promptları **Türkçe** 
- Visual prompt **İngilizce**
- Executive summary promptu **İngilizce** ama Türkçe çıktı bekliyor

Bu, LLM'in bazen İngilizce, bazen Türkçe yanıt vermesine neden olabilir.

**Çözüm**: Visual prompt'un giriş directive'ini de Türkçe yap veya açıkça "Write the prompt **in English** for the image model" de.

### 5. Prompt Uzunluğu ve Token Verimliliği

**Sorun**: `buildAlternativeGenerationPrompt` tek seferde ~800+ token'lık instruction gönderiyor:
- JSON şema örneği (büyük)
- quantityData (tam JSON dump)
- scopeSelections (tam JSON dump)
- DNA locks (tam JSON dump)
- + birçok tekrarlayan instruction

**Çözüm**: 
- JSON şemalarını system prompt'a taşı
- Sadece "delta" veriyi gönder (tüm quantity datasını değil, sadece in-scope olanları)
- DNA locks'u özet string'e dönüştür: `"Duvar: improve/koru/koru/koru | Tavan: improve/esnet/esnet/esnet"`

### 6. Image Prompt'ta Geometri Koruma Çelişkisi

**Sorun**: Prompt "Keep everything unchanged" diyor ama aynı zamanda "redesign these elements" diyor. Image model'ler bu çelişkiyi kötü parse edebilir.

**Çözüm**: Prompt'u 3 net bölüme ayır:
1. **PRESERVE** (değişmeyecekler — liste)
2. **MODIFY** (değişecekler — detaylı)
3. **STYLE** (genel ton/atmosfer)

### 7. `renderPrompt` vs `visualPrompt` vs `designIntentPrompt` Karışıklığı

**Sorun**: 3 farklı "prompt" alanı var ama hangisinin nerede kullanıldığı belirsiz:
- `designIntentPrompt`: Türkçe, kullanıcıya görünür
- `renderPrompt`: Pipeline'dan gelir, `visualPrompt` olarak saklanır
- `visualPrompt`: Hem pipeline'dan hem fallback'ten gelebilir

Image üretiminde bazen `visualPrompt` kullanılırken bazen `buildControlledVisualPrompt()` çağrılıyor.

**Çözüm**: Tek bir `renderReadyPrompt` alanı kullan, pipeline'ın son ürünü her zaman bu olsun.

### 8. briefOptionPools Heuristiğinin Zayıflığı

**Sorun**: `buildMockBriefOptionPools` sadece `planName`'de "ofis/merkez/resepsiyon" arıyor:
```js
const planSuggestsCorporate = /ofis|genel|merkez|resepsiyon/.test(planName);
```
"konferans" kelimesi hiç kontrol edilmiyor → yanlış seçenekler sunulur.

**Çözüm**: Regex'e konferans salonu tiplerini ekle:
```js
const planSuggestsCorporate = /ofis|genel|merkez|resepsiyon|konferans|salon|toplanti/.test(planName);
```
Ve mekan tipine göre farklı option pool'ları sun (konferans → akustik odaklı seçenekler ağırlıklı).
